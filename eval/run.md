# Autoresearch: /infostyle

> Подход Карпати: constraint + mechanical metric + autonomous iteration = compounding gains.

## Установка

```bash
cd /Users/artgas/Desktop/infostyle-skill
```

## Три файла (как у Карпати)

| Файл | Роль | Кто меняет |
|------|------|-----------|
| `SKILL.md` | «train.py» — инструкции скилла | Агент |
| `eval/run.md` | «program.md» — инструкции для агента | Человек |
| `eval/criteria.md` + `eval/test-cases.md` | «prepare.py» — данные и метрика | Неизменяемые |

## Метрика

**Score = количество пройденных критериев / 70 * 100**

10 кейсов × 7 критериев = 70 максимум. Цель: 90%+.

Критерии — механические (grep, подсчёт слов, наличие цифр). НЕ субъективные.

## Процесс одной итерации (~5 минут)

### 1. Прогон (2 мин)
Прочитай `SKILL.md` и `references/`. Для каждого из 10 кейсов из `eval/test-cases.md`:
- Отредактируй текст, следуя SKILL.md
- Запиши результат

### 2. Оценка (1 мин)
Для каждого результата проверь 7 критериев из `eval/criteria.md`.
Посчитай score. Запиши в лог.

### 3. Анализ (1 мин)
- Какие критерии провалены чаще?
- На каких кейсах хуже?
- Что в SKILL.md вызывает ошибку?

### 4. Одно изменение (1 мин)
Сделай ОДНО атомарное изменение в `SKILL.md`:
- Уточни правило
- Добавь пример
- Переформулируй инструкцию
- Добавь предупреждение

**Ограничения:**
- Только `SKILL.md` (НЕ reference-файлы)
- Максимум +10 строк за итерацию
- Не удалять существующие правила
- Не менять формат вывода

### 5. Коммит
```bash
git add SKILL.md
git commit -m "experiment: [описание изменения] score: [X]%"
```

Если score упал → `git revert HEAD` и попробовать другое.

### 6. Повтор
Вернуться к шагу 1. Повторять до прерывания или N итераций.

## Лог результатов

Веди таблицу в `eval/results.tsv`:

```
iteration	score	delta	status	description
0	?	0	baseline	initial SKILL.md
1	?	?	keep/discard	[что изменил]
```

## Важные принципы (Карпати)

1. **Одна метрика** — score в процентах. Не «стало лучше» а «было 75%, стало 80%»
2. **Одно изменение** — атомарное. Если не помогло — revert, не накапливай
3. **Git = память** — каждый эксперимент в коммите, неудачные ревертнуты
4. **Механическая проверка** — grep, подсчёт, наличие. Не «мне кажется»
5. **Когда застрял — думай радикальнее** — перечитай контекст, попробуй неочевидное
